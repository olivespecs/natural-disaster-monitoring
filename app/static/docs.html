WebSocket Connection

Live event stream

Connect to the local FastAPI server to receive completed inference events in real time. The dashboard uses the same channel to push completed EONET analyses as soon as the worker finishes them.

ws://localhost:8000/ws/events
Use wss:// only when deploying behind HTTPS.
REST Endpoints
Method Endpoint Description
GET/api/v1/eventsList enriched events with optional filters
GET/api/v1/events/{event_id}Fetch a single enriched event
GET/api/v1/events/geojsonGeoJSON output for Leaflet map markers
GET/api/v1/analytics/summaryAggregated counts, severity, and trends
GET/api/v1/analytics/hotspotsTop high-severity locations
Event Payload

WebSocket messages use a simple wrapper with a nested completed event payload.

{
  "type": "event_completed",
  "data": {
    "event": {
      "id": "EONET_492",
      "title": "Earthquake near Honshu"
    },
    "status": "completed",
    "inference": {
      "risk_level": "HIGH",
      "severity_score": 82.5,
      "confidence": 0.92,
      "impact_narrative": "The event may affect nearby infrastructure and coastal response planning.",
      "recommendations": [
        "Issue situational awareness alerts",
        "Coordinate with local emergency management",
        "Monitor for secondary impacts"
      ]
    }
  }
}
